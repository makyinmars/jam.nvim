1. INTRODUCTION                                                    *jam-intro*

jam.nvim provides a Telescope interface for Spotify search and playback, plus
public YouTube music-video search with a YouTube Music handoff.

Requirements:
  - Neovim 0.10 or newer
  - telescope.nvim
  - curl
  - openssl, Spotify Premium, and an active Spotify Connect device for Spotify
    playback control
  - A YouTube Data API v3 key for YouTube music-video search

Album artwork additionally requires chafa, or image.nvim in a compatible
terminal. Without either renderer, the preview displays the artwork URL.

==============================================================================
2. SETUP                                                            *jam-setup*

Create an application in Spotify's developer dashboard and register this exact
redirect URI:

  http://127.0.0.1:8765/callback

Then configure jam.nvim:
>
  require("jam").setup({
    providers = {
      spotify = {
        client_id = vim.env.SPOTIFY_CLIENT_ID,
      },
    },
  })
<
Run `:checkhealth jam`, then `:Jam auth spotify` to connect your account. Open
Spotify and play something once before using playback controls; merely opening
the app may not make it an active Spotify Connect device.

For YouTube music-video search, enable YouTube Data API v3 for a Google Cloud
project, create an API key restricted to that API, and configure:
>
  require("jam").setup({
    provider = "youtube_music",
    providers = {
      youtube_music = {
        api_key = vim.env.YOUTUBE_API_KEY,
        region_code = "US", -- optional
        relevance_language = "en", -- optional
      },
    },
  })
<
This searches YouTube's public Music video category. It is not a YouTube Music
catalog, library, queue, or playback-control API.

3. COMMANDS                                                      *jam-commands*

                                                          *:Jam*
:Jam                      Open the search picker.
:Jam search               Open the search picker.
:Jam auth spotify         Authenticate with Spotify.
:Jam logout               Remove stored Spotify credentials.
:Jam play                 Resume playback.
:Jam pause                Pause playback.
:Jam next                 Skip to the next item.
:Jam previous             Return to the previous item.
:Jam now-playing          Display the current track.
:Jam health               Run health diagnostics.

Prefix a search with `a:` for albums, `t:` for artists, `s:` for songs and
tracks, `p:` for podcasts, or `e:` for podcast episodes. A query without a
prefix searches all supported item types.

The search picker maps <CR> to open an album, artist, or podcast, or play the
selected track or episode. Albums open in disc and track order, artists open to
their top tracks, and podcasts open to their episodes. In a collection picker,
<CR> plays the selected item and <Esc> returns to the original search. CTRL-Q
queues the selected item and CTRL-P pauses playback.

The YouTube provider uses explicit submit to protect the API's limited search
quota. Enter a query and press CTRL-S, or press <CR> when no result is selected.
The results title shows "Loading…" while the request is in flight. Results are
labelled VIDEO and attributed to YouTube Music. Press CTRL-T to toggle whether
the selected result opens in YouTube Music or regular YouTube; the current
target is shown in the picker title. The configured open_host chooses the
initial target, and YouTube remains the fallback when the Music host fails.
Exact queries are cached for the current Neovim session. YouTube does not map
queue or playback actions.

The picker can also be opened after loading the Telescope extension:
>
  require("telescope").load_extension("jam")
  :Telescope jam
<

4. ARTWORK                                                      *jam-artwork*

With `backend = "auto"`, jam.nvim selects:

  1. image.nvim in a detected compatible terminal
  2. chafa when its executable is available
  3. text metadata

Detection is best effort. Set `artwork.backend` to "image", "chafa", "text", or
"none" to override it. Run `:checkhealth jam` to inspect the selected backend.
